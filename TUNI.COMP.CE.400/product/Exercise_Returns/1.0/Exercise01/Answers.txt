4. How did decreasing the quantization value affect perceived quality of the video output?
Decreasing the quantization value improves the perceived quality of the video output. This is because a lower QP value results in less aggressive compression, which retains more detail and leads to better image quality. On the other hand, a higher QP increases the compression, causing a loss in visual quality as details are removed.

5. How did decreasing the quantization value affect size of the video output file?
	QP 50: 212 KB
	GROUP QP: 906 KB
A lower QP (like QP 50) results in more aggressive compression, producing a smaller file. As the QP value decreases, the file size increases because less compression is applied to retain more detail.

10. What does parameter -pg do in GCC? How it affects the resulting executable and its execution?
The -pg parameter in GCC is used to enable profiling. It generates additional code to record performance data during the execution of the program. This information is typically stored in a file called gmon.out and can be analyzed with tools like gprof. It didn't affect the program's functionality, but it slow down execution slightly because of the overhead introduced by the profiling process.

12. Why it is better to look at percentages rather than absolute time like seconds or milliseconds while using a profiling tool (like gprof)?
Percentages show you which parts of the code are the most time-consuming, regardless of the absolute execution time.And, if we are comparing multiple runs or optimizing a specific function, percentage time gives a more uniform comparison rather than the absolute execution time, which may be influenced by external factors like system load.
